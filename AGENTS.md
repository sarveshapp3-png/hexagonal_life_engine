# Obsidian Vault Rules

This repository contains an Obsidian vault.

## How to Work

Before implementing features:

1. Read relevant notes.
2. Follow documented requirements.
3. Update notes when code changes.

## Creating Notes

Create notes for:

* Systems
* Features
* Architecture
* Decisions
* Roadmaps

Use Markdown.

## Linking

Always use Obsidian links.

Example:

[[Evolution]]

[[Neural Network]]

[[Physics]]

Link related notes together.

## Folder Structure

Organize notes by topic.

Example:

* Overview
* Systems
* Gameplay
* UI
* Architecture
* Roadmap

## Updating Documentation

When code changes:

* Update relevant notes.
* Update architecture documentation.
* Update feature documentation.

Documentation must remain synchronized with code.

## Goal

The Obsidian vault should be complete enough that a developer can understand the project without reading the codebase.

Treat the Obsidian vault as project documentation and a source of truth.
